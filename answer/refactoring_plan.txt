      グローバル変数リファクタリング提案
      
      ## 目的
      現在、複数のJavaScriptファイルでグローバルスコープが汚染されており、変数の衝突や意図しない上書きが発生するリスクがあります。
      この問題を解決するため、すべてのグローバル変数と関数を唯一のグローバルオブジェクト `App` の配下に集約し、各ファイルを即時実行関数（IIFE）で囲むことで、ファイルごとのスコープを独立させます。
      
      ## 全体方針
      1.  **グローバル名前空間の作成**: HTMLで最初に読み込むJavaScriptファイル（または `shakkou_donperi_core.js` の冒頭）で `const App = {};` を定義します。
      2.  **スコープの分離**: 各JavaScriptファイルのコード全体を `(function(app) { /* ... */ })(App);` という即時実行関数で囲みます。
      3.  **機能のモジュール化**: 各ファイルが公開する変数や関数は、`app` オブジェクト（`App` グローバルオブジェクトへの参照）のプロパティとして定義します。ファイル内部でのみ使用する変数や関数は、IIFEのローカル変数としてカプセル化します。
      
      ---
      
      ## ファイル別リファクタリング案
      
      ### 1. `shakkou_donperi_core.js`
      このファイルをアプリケーションのコアとし、`App` オブジェクトの初期化と主要なロジックの定義を行います。
      
      *   ファイルの先頭で `const App = {};` を宣言し、アプリケーションの唯一のグローバル名前空間を定義します。
      *   ファイル全体のコードを即時実行関数 `(function(app) { ... })(App);` で囲みます。
      *   現在グローバルスコープに定義されている計算ロジック関数（`calculateSenkou`, `calculateMakuri`, `calculateSashi`, `adjustForStrategy` など）を、`app.calculateSenkou = function(...) {}` のように `App` オブジェクトのメソッドとして再定義します。
      *   `getBankData` 関数も `app.getBankData = function(...) {}` とし、取得したバンクデータは `app.bankData` のように `App` オブジェクトに格納します。
      *   他のファイルから参照される必要のある変数（例: 計算結果を保持する変数）も、すべて `app` オブジェクトのプロパティとして定義します。
      
      ### 2. `shakkou_donperi_display.js`
      UIの更新とユーザー操作のハンドリングを担当します。コアロジックには依存しますが、自身のロジックをグローバルに公開する必要はありません。
      
      *   ファイル全体のコードを即時実行関数 `(function(app) { ... })(App);` で囲みます。
      *   `shakkou_donperi_core.js` で定義された `app.calculate...` や `app.bankData` などの `App` オブジェクトのプロパティを参照して、UIの表示や更新を行います。
      *   `DOMContentLoaded` イベントリスナーや、ボタンクリックなどのイベントハンドラは、IIFE内で定義します。これにより、イベントハンドラ内の処理もグローバルスコープを汚染しません。
      *   UI更新のための関数（例: `displayResults`）は、他のファイルから直接呼び出される必要がなければ、IIFE内のローカル関数として定義します。
      
      ### 3. `keirin_logic.js`
      `shakkou_donperi_core.js` とは異なる、独自の競輪関連ロジックを扱います。他のファイルと同様にリファクタリングを行います。
      
      *   ファイル全体のコードを即時実行関数 `(function(app) { ... })(App);` で囲みます。
      *   公開する必要のある関数や変数は、`app` オブジェクトのプロパティとして定義します。例えば、`app.keirin = app.keirin || {};` のように専用の名前空間を設けることも有効です。
      
      ### 4. `meigen.js`
      ランダムな格言を表示する自己完結型の機能です。
      
      *   ファイル全体のコードを即時実行関数 `(function(app) { ... })(App);` で囲みます。
      *   格言リスト `maxims` は、他のファイルから参照する必要がないため、IIFE内のローカル変数 `const maxims = [...]` として定義し、カプセル化します。
      *   格言を表示するロジック（`displayRandomMaxim` 関数など）もIIFEのローカル関数とし、DOM読み込み完了時に自動実行されるように設定します。
      *   `App` オブジェクトへのプロパティ追加は、他のファイルとの連携がなければ不要です。
      
      ### 5. `paste-input.js`
      入力欄への貼り付け機能を提供する、自己完結型の機能です。
      
      *   ファイル全体のコードを即時実行関数 `(function(app) { ... })(App);` で囲みます。
      *   イベントリスナーのセットアップロジックはすべてIIFE内で完結させます。
      *   この機能は他のファイルと直接連携する必要がないため、`App` オブジェクトへのプロパティ追加は不要です。
      
      ### 6. `tamaki_speech.js` と `kiyone_voice.js`
      キャラクターの発話や音声に関する機能です。キャラクターごとにモジュールを整理します。
      
      *   各ファイル全体のコードを即時実行関数 `(function(app) { ... })(App);` で囲みます。
      *   キャラクターに関連する機能（セリフデータ、再生関数など）を、`App` オブジェクトの下にキャラクターごとの名前空間を作成して格納します。
          *   **例 (`tamaki_speech.js`):**
              ```javascript
              // App.tamaki オブジェクトがなければ作成
              app.tamaki = app.tamaki || {};
      
              // セリフデータを格納
              app.tamaki.speeches = { normal: [...], smile: [...] };
      
              // 再生関数を定義
              app.tamaki.playSpeech = function(emotion) { ... };
              ```
          *   **例 (`kiyone_voice.js`):**
              ```javascript
              // App.kiyone オブジェクトがなければ作成
              app.kiyone = app.kiyone || {};
      
              // 音声再生関数を定義
              app.kiyone.playVoice = function() { ... };
              ```
      *   これにより、`App.tamaki.playSpeech('smile')` のように、外部から安全に関数を呼び出せるようになります。
